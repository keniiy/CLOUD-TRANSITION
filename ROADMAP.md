# CLOUD TRANSITION Roadmap

## Goal

Transition from backend/platform-heavy software engineering into Cloud Engineering through hands-on, project-based learning.

The project will grow from Linux fundamentals into a full mixed-language cloud platform supporting TypeScript and Python services.

## Target Roles

- Cloud Engineer
- Platform Engineer
- DevOps Engineer
- AWS Engineer
- Infrastructure Engineer
- Backend Engineer, Cloud Platform
- Site Reliability Engineer

## Stage-Gated Rule

I only move to the next stage when I can:

1. Explain the concept
2. Build a working version
3. Break and fix common issues
4. Document the work
5. Create CV/LinkedIn/interview material from it

## Stages

### Stage 0: Project Setup

Set up the CLOUD TRANSITION workspace, roadmap, progress tracker, and GitHub repo.

### Stage 1: Linux Server Fundamentals

Deploy and manage a simple TypeScript/Node.js API on a Linux server.

### Stage 2: Networking, Nginx, DNS, and HTTPS

Expose the API safely using reverse proxy, domain routing, and HTTPS.

### Stage 3: AWS Foundation

Secure AWS account, deploy EC2-hosted API, configure IAM, security groups, S3, CloudWatch, and cost controls.

### Stage 4: RDS and S3 Integration

Connect the API to managed PostgreSQL and object storage.

### Stage 5: Docker for Node and Python Services

Containerize TypeScript and Python services with Docker and Docker Compose.

### Stage 6: CI/CD with GitHub Actions

Automate testing, building, and deployment.

### Stage 7: Load Balancing and Scaling

Deploy multiple service instances behind a load balancer with health checks.

### Stage 8: Python Cloud Automation

Use Python and boto3 to inspect, automate, and report on cloud resources.

### Stage 9: Terraform Basics

Provision basic AWS resources using Infrastructure as Code.

### Stage 10: Terraform Production Infrastructure

Provision VPC, subnets, ALB, compute, RDS, CloudWatch, IAM, and security groups with Terraform modules.

### Stage 11: ECS Fargate

Deploy Dockerized TypeScript and Python services using AWS ECS Fargate.

### Stage 12: Kubernetes Fundamentals

Run the services locally on Kubernetes using deployments, services, ingress, configmaps, secrets, and probes.

### Stage 13: Monitoring and Observability

Add CloudWatch logs, Prometheus metrics, Grafana dashboards, alerts, and incident runbooks.

### Stage 14: Cloud Security

Harden IAM, secrets, security groups, RDS, S3, TLS, and dependency management.

### Stage 15: Platform Engineering Templates

Create reusable service templates for Node.js and Python services with shared Docker, CI/CD, health checks, metrics, and deployment standards.

### Stage 16: Capstone Multi-Service Cloud Platform

Build a production-style cloud platform that deploys and operates both TypeScript and Python services on AWS.

### Stage 17: LinkedIn, CV, and Applications

Convert the project into LinkedIn posts, CV bullets, recruiter messages, interview stories, and job applications.
